# 销售系统 - 需求规格说明书

## 一、系统概述

销售系统用于管理商品销售业务，支持不同角色的用户（管理员、店长、销售员）进行相应的操作。

## 二、用户角色与权限

### 1. 管理员
- 自身密码修改
- 用户信息管理：添加、修改、查询、删除
- 商品信息管理：添加、修改、查询、删除
- 销售报表显示：日报表、月报表、商品销售量报表、销售员业绩报表
- 退出登陆系统

### 2. 店长
- 自身密码修改
- 商品信息管理：添加、修改、查询、删除
- 销售报表显示：日报表、月报表、商品销售量报表、销售员业绩报表
- 退出登陆系统

### 3. 销售员
- 商品查询浏览、商品销售
- 自己销售报表显示：日报表、月报表
- 退出登陆系统

## 三、功能模块

### 3.1 登录模块
- 用户登录验证
- 密码验证
- 根据用户角色显示不同菜单

### 3.2 用户管理模块（仅管理员）
- 添加用户：用户名、密码、角色
- 修改用户：修改用户名、密码、角色
- 查询用户：按用户名查询
- 删除用户：删除指定用户

### 3.3 商品管理模块（管理员、店长）
- 添加商品：商品编号、商品名称、单价、库存量
- 修改商品：修改商品信息
- 查询商品：按商品编号或名称查询
- 删除商品：删除指定商品

### 3.4 销售模块（销售员）
- 商品查询浏览：查看所有商品信息
- 商品销售：选择商品、输入数量、生成销售记录

### 3.5 报表模块

#### 日报表（管理员、店长、销售员）
- 显示当日所有销售记录
- 销售员只能查看自己的记录

#### 月报表（管理员、店长、销售员）
- 显示当月所有销售记录
- 销售员只能查看自己的记录

#### 商品销售量报表（管理员、店长）
- 统计每个商品的销售数量和金额

#### 销售员业绩报表（管理员、店长）
- 统计每个销售员的销售业绩

### 3.6 密码修改模块（所有用户）
- 修改自己的登录密码

## 四、数据结构

### 4.1 用户信息文件
```
用户ID | 用户名 | 密码 | 角色
```
角色：1-管理员，2-店长，3-销售员

### 4.2 商品信息文件
```
商品编号 | 商品名称 | 单价 | 库存量
```

### 4.3 销售记录文件
```
销售ID | 商品编号 | 商品名称 | 销售数量 | 单价 | 总金额 | 销售日期 | 销售员ID
```

### 4.4 密码文件
```
管理员密码
```

## 五、界面设计

### 5.1 主菜单
```
========================================
          销售系统
========================================
1. 修改密码
2. 用户管理（仅管理员）
3. 商品管理
4. 销售商品（仅销售员）
5. 日报表
6. 月报表
7. 商品销售量报表（管理员、店长）
8. 销售员业绩报表（管理员、店长）
0. 退出
========================================
```

### 5.2 子菜单格式
各个功能模块采用子菜单形式，提供清晰的操作选项。

## 六、技术要求

### 6.1 C语言控制台版
- 文本菜单界面
- 文件存储数据
- 菜单选择超过9用字符代替（A、B、C...）

### 6.2 PyQt5桌面版
- 图形界面
- 侧边栏导航
- 表格数据显示

### 6.3 Streamlit网页版
- 网页界面
- 侧边栏导航
- 表格和图表展示
