# 俄罗斯方块 AI 助手

基于 Q-Learning 强化学习算法的俄罗斯方块智能托管助手，支持人类干预训练。

## 文件说明

- `tetris_ai.h` - AI 头文件，包含函数声明和数据结构
- `tetris_ai.c` - AI 实现，包含 Q-Learning 算法
- `imitation_learning.h` - 模仿学习头文件
- `imitation_learning.c` - 模仿学习实现
- `main.c` - 主程序，集成 AI 和人类干预功能
- `tetris_qtable.txt` - 训练好的 Q 值表
- `tetris_player_records.txt` - 人类玩家游戏记录

## 编译

使用统一的编译脚本：

```powershell
.\compile.bat
```

脚本会编译所有模块并生成：
- `main.exe` - 主游戏程序（包含 AI 观看、训练模式）
- `train_ai.exe` - 独立训练程序

## 使用方法

### 1. 运行游戏

```powershell
.\main.exe
```

### 2. 主菜单选项

- **0. 退出游戏**
- **1. 单人游戏** - 手动游玩，可记录操作用于模仿学习
- **2. AI 观看模式** - AI 自动游玩，观察 AI 决策
- **3. AI 训练模式** - AI 自我训练，支持人类干预
- **4. 查看历史战绩** - 显示历史得分记录
- **5. 从玩家记录初始化 AI** - 基于人类游玩记录初始化Q表

### 3. AI 训练模式（带人类干预）

进入训练模式后：
- 按 **P** - 暂停/继续
- 按 **Q** - 停止训练
- 按 **H** - 开启/关闭人类干预训练

#### 人类干预机制

当开启人类干预（按H）后：

1. **AI 先决策** - 每个新方块出现时，AI 先给出初始决策
2. **显示 AI 决策** - 控制台显示 `[AI决策] 当前:I 下一个:T 高度:X 空洞:X 不平:X -> 旋转:R 位置:P`
3. **人类可修正** - 使用方向键调整方块：
   - **上键** - 旋转（带墙踢支持）
   - **左/右键** - 左右移动
   - **下键** - 加速下落

AI 学习人类修正后的效果，通过奖励反馈改进策略。

### 4. 启发式知识注入

为了加速AI训练，系统内置了启发式知识库：

#### 4.1 自动注入机制

当AI从头开始训练时（没有加载已有Q表），系统会自动：
1. 遍历所有可能的游戏状态（共4900种状态）
2. 对每个状态构建代表性棋盘
3. 使用启发式评估算法预计算每个动作的价值
4. 将启发式值注入到Q表作为初始值

这个过程让AI从一开始就具备基本的游戏理解能力，大大减少探索时间。

#### 4.2 启发式评估内容

启发式评估包含以下知识：

**1. 方块形状理解**
- 7种基本方块形状（I, J, L, O, S, T, Z）
- 每种方块的4种旋转状态
- 方块尺寸（宽度和高度）
- 方块特性（如I方块适合消多行，O方块适合平铺）

**2. 位置契合度评估**
- 分析落点位置的列高度分布
- 评估方块与下方结构的匹配程度
- 考虑不平整表面的适应性

**3. 放置效果预测**
- 消除行数预测（优先消多行）
- 高度变化评估（保持低堆叠）
- 空洞产生风险检测（避免新空洞）
- 不平整度影响分析
- 填补接近完成行的潜力

**4. 前瞻策略**
- 评估下一个方块的可放置性
- 为未来留出更多移动空间
- 平衡当前收益和未来潜力

**5. 游戏阶段适应**
- 早期：优先铺好底层
- 中期：保持平稳，减少空洞
- 后期：优先消除，避免堆满

**6. 特定方块偏好**
- I方块：奖励消除2行以上的放置
- T/L/J方块：奖励填补空洞的放置
- O方块：奖励在平坦区域的放置
- S/Z方块：奖励消除行的放置

## 算法说明

### Q-Learning 基础

Q-Learning 是一种无模型强化学习算法，通过学习状态-动作值函数 Q(s,a) 来选择最优策略。

**更新公式：**
```
Q(s,a) = Q(s,a) + α [r + γ max Q(s',a') - Q(s,a)]
```

其中：
- α: 学习率（AI模式 0.05，人类干预时 0.5）
- γ: 折扣因子（0.95，关注长期奖励）
- r: 奖励
- s: 当前状态
- a: 当前动作
- s': 下一状态
- a': 下一状态的最佳动作

### 状态特征

- `aggregate_height` - 总高度（越低越好）
- `complete_lines` - 消除行数（越多越好）
- `holes` - 空洞数量（越少越好）
- `bumpiness` - 表面不平整度（越小越好）
- `highest_point` - 最高点（越低越好）
- `near_complete_lines` - 近似完成的行数（填补缺口策略）
- `deepest_line_gap` - 最大行缺口深度（用于评估填充策略）

### 状态空间设计

**基于特征的新状态空间：**

状态包含以下信息：
- 当前方块类型 (0-6)
- 下一个方块类型 (0-6)
- 高度等级 (0-4)：离散化后的总高度
- 空洞等级 (0-3)：离散化后的空洞数量
- 不平整度等级 (0-4)：离散化后的表面不平整度

**特征离散化表：**

| 特征 | 等级0 | 等级1 | 等级2 | 等级3 | 等级4 |
|------|-------|-------|-------|-------|-------|
| 高度 | 0-3 | 4-7 | 8-11 | 12-15 | 16+ |
| 空洞 | 0 | 1-3 | 4-6 | 7+ | - |
| 不平整度 | 0-3 | 4-7 | 8-11 | 12-15 | 16+ |

**状态索引计算：**
```
shape_state = 当前方块 × 7 + 下一个方块         (0-48)
feature_state = 高度等级 × 20 + 空洞等级 × 5 + 不平整度等级  (0-99)
状态索引 = shape_state × 100 + feature_state    (0-4899)
```

**总状态数：**
- 方块组合：7 × 7 = 49
- 特征组合：5 × 4 × 5 = 100
- 总状态数：49 × 100 = **4900种状态**
- 每种状态动作数：40（4旋转 × 10位置）
- Q表大小：4900 × 40 = **196,000条目**

### 奖励函数设计

**基础奖励（calculate_reward）：**
```
奖励 = 消除行数 × 100
     - (放置后高度 - 放置前高度) × 3.0
     - (放置后空洞 - 放置前空洞) × 5.0
     - (放置后不平整 - 放置前不平整) × 1.0
```

**额外奖励（人类修正时）：**
```
奖励 += 消除行数 × 250
奖励 += 降低高度 ? 50 : 0
奖励 += 减少空洞 ? 30 : 0
奖励 += 填补缺口 × 40
```

**人类干预学习反馈：**

- **修正AI错误** - AI决策导致负奖励，人类修正后为正奖励
  - 显示：`[修正AI错误] 原奖励-50.0 -> 修正后80.0 (净提升130.0)`
  - AI学到"这个状态下不要这样决策"

- **补充AI能力** - AI决策还行，人类微调后更好
  - 显示：`[补充AI能力] 原奖励20.0 -> 微调后100.0 (提升80.0)`
  - AI学到"这个状态下可以做得更好"

- **修正未改进** - 人类修正反而更差
  - 显示：`[修正未改进] 原奖励50.0 -> 修正后30.0`
  - AI学到"我的决策是对的"

### 模仿学习

从人类玩家记录中学习：

```c
// 加载人类玩家记录
initialize_q_from_records(&ai_agent, "tetris_player_records.txt");
```

**学习策略：**
1. 解析记录中的状态-动作对
2. 根据动作效果评估每个记录的价值
3. 相对优势评估（好动作×1.5，坏动作×0.8）
4. 移除频率偏差，避免过拟合

**记录格式：**
```
当前方块 下一个方块 旋转 位置 高度等级 空洞等级 不平整度等级 消除行数 前置分数 后置分数 后置高度 后置空洞 后置不平整
```

### 墙踢（Wall Kick）

支持在狭窄空间中旋转：

**测试顺序：**
1. 原地旋转
2. 左移1格旋转
3. 右移1格旋转
4. 左移2格旋转
5. 右移2格旋转
6. 左移1格、下移1格旋转
7. 右移1格、下移1格旋转

## 训练参数

**在 tetris_ai.c 的 init_ql_agent() 中可以调整：**

```c
learning_rate = 0.05;      // 学习率（基础训练）
discount_factor = 0.95;    // 折扣因子
exploration_rate = 0.3;     // 探索率
```

**人类干预时的学习率**：
- 人类干预修正使用更高的学习率（0.5）快速学习
- AI自主训练时使用基础学习率（0.05）渐进优化

### 决策机制

#### 1. 人类干预模式

每个方块只在生成后的第一个下落周期做一次决策：
- AI给出初始建议
- 人类可以修正AI的决策
- 如果人类没有干预，AI按原计划执行
- 适合"教学式训练"

#### 2. 自主训练模式（无人类干预）

AI可以在每次下落周期重新评估并调整决策：
- 动态适应游戏状态变化
- 实时优化决策策略
- 显示标记为 `(持续重估)`
- 适合"自主探索优化"

### 初始化策略

**三种初始化方式：**

1. **从文件加载**（优先）
   - 加载已有的训练数据
   - 立即使用学习到的策略

2. **启发式初始化**（默认）
   - 当没有已有训练数据时自动触发
   - 注入专家级启发式知识
   - 让AI从一开始就具备基本游戏理解能力
   - 大大减少训练时间

3. **模仿学习初始化**
   - 从人类玩家记录中学习
   - 主菜单选项5触发
   - 适合有大量人类游玩记录的场景

**参数含义：**
- **学习率**：越大学习越快，但可能不稳定；值太小则学习缓慢
- **折扣因子**：越大越关注长期回报，越小越关注即时奖励
- **探索率**：越大越随机探索新策略，越小越依赖已知策略

**防止Q值爆炸：**
```c
// 限制Q值在 [-10000, 10000] 范围内
if (new_q > 10000.0) new_q = 10000.0;
if (new_q < -10000.0) new_q = -10000.0;
```

## 性能优化建议

1. 使用更小的状态空间减少 Q 表大小
2. 使用函数近似（如神经网络）代替 Q 表
3. 使用经验回放缓冲区
4. 添加优先经验采样
5. 使用深度 Q 网络 (DQN)

## 注意事项

- 初次训练时启发式初始化会花费一些时间（约10-30秒）
- 训练质量取决于奖励函数设计
- 人类干预训练比自动训练效果更好
- 墙踢功能支持在极端情况下旋转
- 可以用预训练的 Q 表快速开始
- 每10回合自动保存训练数据到 `tetris_qtable.txt`

## 人类玩家记录

手动游戏时可以记录操作，保存到 `tetris_player_records.txt`：

**记录条件：**
- 游戏得分 >= 12000 分
- 玩家同意保存

**记录用途：**
- AI模仿学习人类的策略
- 快速初始化Q表，避免从零开始学习

## 训练加速技巧

1. **启用启发式初始化**（默认已启用）
   - 让AI从基础理解开始学习
   - 比完全随机初始化快5-10倍

2. **人类干预训练**
   - AI给出初始建议
   - 人类修正明显错误的决策
   - 更高的学习率快速吸收人类经验

3. **合理设置训练回合数**
   - 启发式初始化后，50-100回合即可看到明显效果
   - 200-500回合可以训练出不错的策略
   - 1000+回合可以达到较高水平

4. **分阶段训练**
   - 早期：让AI自由探索，建立基本策略
   - 中期：开启人类干预，纠正明显错误
   - 后期：关闭干预，让AI自主优化
