# 出勤管理系统 - 功能演示文档

## 项目概述

出勤管理系统用于企业职工的出勤管理，支持打卡记录、出勤计算、请假管理、统计分析等功能。系统通过读卡机记录职工上班时间，自动计算迟到次数、旷工天数，并根据企业规定计算奖罚金额。

## 三个版本对比

### 1. C控制台版
- **技术栈**：C语言
- **界面**：命令行菜单
- **特点**：
  - 轻量级，无需安装额外依赖
  - 适合Windows/Linux环境
  - 快速响应，资源占用少

### 2. PyQt5桌面版
- **技术栈**：Python + PyQt5
- **界面**：图形化窗口
- **特点**：
  - 现代化GUI界面
  - 侧边栏导航，操作直观
  - 表格数据展示清晰

### 3. Streamlit网页版
- **技术栈**：Python + Streamlit + Pandas
- **界面**：Web浏览器
- **特点**：
  - 跨平台，通过浏览器访问
  - 数据可视化展示
  - 响应式布局

## 核心功能演示

### 1. 登录验证
- 默认密码：`admin123`
- 密码存储在 `dataset/password.txt`
- 三次尝试机会

### 2. 打卡记录处理
- 读取当日打卡记录文件 `punchIn.txt`
- 自动计算迟到次数（每迟到10分钟记1次）
- 自动判断旷工（未打卡或迟到超过1小时）
- 更新月出勤情况文件 `Attendance.txt`
- 追加到月打卡记录文件 `MonthPunchIn.txt`

**示例数据**：
```
E001 7 55 30  # 正常上班
E002 8 5 45   # 迟到5分钟，记1次
E003 9 10 30  # 迟到超过1小时，记旷工
```

### 3. 查询功能

#### 按职工编号查询
- 显示职工基本信息
- 显示当日到厂时间
- 显示当月出勤情况（迟到次数、请假天数、旷工天数）

#### 全勤职工清单
- 列出所有全勤职工
- 条件：迟到次数=0 且 请假天数=0 且 旷工天数=0

#### 请假天数超过5天的职工
- 列出请假天数超过5天的职工
- 扣除允许请假天数后计算
- 显示允许天数、实际天数、超出天数

#### 旷工职工查询
- 列出所有有旷工行为的职工
- 显示完整的出勤信息

### 4. 请假管理
- 输入职工编号进行请假登记
- 请假天数自动累加
- 如果有旷工天数，自动减1（请假不记旷工）

**请假优惠规则**：
- 女职工每月可请假3天
- 55岁以上男职工每月可请假2天
- 生日所在月份可请假1天

### 5. 统计功能
- 职工总数
- 当月全勤职工数及百分比
- 当日最长迟到时间
- 当月最大请假天数
- 当月最大迟到次数
- 当月最大旷工天数
- 当日迟到职工数及百分比

### 6. 排序功能

#### 最早到厂的职工
- 查找当日最早的打卡记录
- 显示职工编号、姓名、到厂时间

#### 按出生日期升序输出全体职工
- 按出生日期从小到大排序
- 显示完整的职工信息

#### 按出生日期降序列出全体女职工
- 筛选女职工
- 按出生日期从大到小排序

## 数据文件说明

### EmpBasic.txt（职工基本信息）
格式：`编号 姓名 性别 出生日期 职务级别 所在部门`
```
E001 张三 男 19850211 中级 技术部
E002 李四 女 19900305 高级 财务部
```

### Attendance.txt（月出勤情况）
格式：`编号 迟到次数 请假天数 旷工天数`
```
E001 2 1 0
E002 0 3 1
```

### punchIn.txt（当日出勤记录）
格式：`编号 时 分 秒`
```
E001 7 55 30
E002 8 5 45
```

### MonthPunchIn.txt（月逐日出勤情况）
格式：`编号 时 分 秒`
```
E001 7 55 30
E001 8 0 15
```

## 核心算法

### 迟到次数计算
```
到达时间 = 时 * 3600 + 分 * 60 + 秒
标准时间 = 8 * 3600 = 28800

迟到时间 = 到达时间 - 标准时间

如果 迟到时间 > 0:
    迟到分钟数 = 迟到时间 / 60
    迟到次数 = ceil(迟到分钟数 / 10)
```

### 旷工判断
```
旷工条件（满足其一即为旷工）：
1. 当日出勤文件中不出现该职工编号
2. 迟到时间 >= 3600（迟到1小时及以上）
```

### 允许请假天数计算
```
allowed_leave_days = 0

如果 性别 == "女":
    allowed_leave_days += 3

如果 性别 == "男" 且 年龄 >= 55:
    allowed_leave_days += 2

如果 当前月份 == 生日月份:
    allowed_leave_days += 1
```

### 年龄计算
```
current_year = 当前年份
current_month = 当前月份
current_day = 当前日期

birth_year = 出生日期的前4位
birth_month = 出生日期的5-6位
birth_day = 出生日期的7-8位

age = current_year - birth_year

if current_month < birth_month or (current_month == birth_month and current_day < birth_day):
    age -= 1
```

## 运行指南

### C控制台版
```bash
cd Attendance_Management_System/c_console
compile.bat    # Windows编译
main.exe        # 运行
```

### PyQt5桌面版
```bash
cd Attendance_Management_System/pyqt5_gui
pip install -r requirements.txt
run.bat        # Windows
python main.py # 跨平台
```

### Streamlit网页版
```bash
cd Attendance_Management_System/streamlit_app
pip install -r requirements.txt
run.bat              # Windows
streamlit run app.py # 跨平台
```

访问地址：http://localhost:8501

## 开发总结

本项目成功实现了出勤管理系统的三个版本，涵盖了以下技术点：

1. **数据结构设计**：职工信息、出勤记录、打卡记录
2. **文件I/O操作**：读写文本文件，数据持久化
3. **算法实现**：迟到计算、旷工判断、请假优惠、年龄计算
4. **用户界面**：命令行、GUI、Web三种界面风格
5. **数据验证**：登录验证、日期格式验证
6. **统计分析**：多项统计指标计算
7. **排序功能**：多种排序方式

三个版本各有特色，满足不同场景的使用需求。
