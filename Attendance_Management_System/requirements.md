# 出勤管理系统 - 详细需求规格说明书

## 一、项目背景

在计算机化管理的企业中，每人都配有一张磁卡或IC卡。卡中包括职工编号、姓名、所在部门、职务等信息。其中职工编号是唯一的。

本题目仅涉及有关出勤卡数据处理部分。

## 二、业务规则

### 2.1 系统规定

**读卡机规定：**
- 每个职工每天早晨上班时要在读卡机上刷卡
- 一旦刷卡完毕，该职工的编号和刷卡时间就形成一个记录存入出勤情况 `punchIn` 文件中
- 正点上班时间为早上8点，早上9点准时关闭读卡机
- 假设该企业最多有500名职工

**数据文件：**
- `EmpBasic` - 职工基本信息文件（职工编号、姓名、性别、出生日期、职务级别、所在部门）
- `Attendance` - 职工月出勤情况文件（职工编号、迟到次数、请假天数、旷工天数）
- `punchIn` - 当日出勤记录文件（职工编号、时、分、秒）
- `MonthPunchIn` - 月逐日出勤情况文件

### 2.2 奖罚规定

**旷工扣款：**
- 每旷工一天就扣发当月工资的10%
- 旷工定义：当日出勤文件中不出现的职工（编号）或迟到1小时及以上的职工

**迟到扣款：**
- 迟到10分钟以下：记录迟到1次
- 迟到10-20分钟（不含20）：记录迟到2次
- 迟到20-30分钟（不含30）：记录迟到3次
- 以此类推，每迟到一次就扣发当月工资的0.5%

**请假扣款：**
- 对于超过允许请假天数之外的，每请假1天扣发当月工资的5%

**请假优惠：**
- 女职工每月可请假3天（免扣工资）
- 55岁以上的男职工每月可请假2天（免扣工资）
- 每个职工生日所在月份可请假1天（免扣工资）

**全勤奖励：**
- 全勤的职工一次性奖励400元（只要有请假就不算全勤）

## 三、系统功能要求

### 3.1 数据读取要求

1. 只有关闭读卡机后才能读取 `punchIn` 文件中的数据
2. 当读取 `punchIn` 文件时，要求同时记录职工的迟到次数、旷工天数并累计写入月出勤情况文件中
3. `punchIn` 文件的内容为职工编号、时、分、秒，每个数据项均是整数类型

### 3.2 职工基本信息要求

4. 为简化设计，职工基本信息文件中仅含职工编号、姓名、性别、出生日期

### 3.3 查询功能（菜单形式）

程序要用菜单形式提供以下查询功能：

1. **按职工编号查询**
   - 查询当日到厂时间
   - 查询当月出勤情况（列出该职工迟到次数、请假天数、旷工天数）

2. **列出所有全勤职工的清单**
   - 包含职工编号、姓名、性别、出生日期

3. **列出请假天数超过5天的职工情况**
   - 含职工编号、姓名、性别、出生日期、请假天数
   - 请假天数指减去允许请假天数后的天数

4. **列出有旷工行为的全体职工的情况**
   - 含职工编号、姓名、性别、出生日期、迟到次数、请假天数、旷工天数

### 3.4 请假管理（菜单形式）

6. 用菜单形式输入请假职工的编号
   - 输入后要把请假一天的信息写入月出勤文件中
   - 与原请假天数累计
   - 同时从旷工天数中减1（如果有旷工天数的话）

### 3.5 统计功能（菜单形式）

7. 用菜单形式提供以下统计功能：
   - 计算当前职工总数
   - 计算当月全勤职工总数及占全体职工的百分比
   - 计算当日最长的迟到时间
   - 计算当月最大的请假天数
   - 计算当月最大的迟到次数
   - 计算当月旷工的最大天数
   - 计算当日迟到的职工总数及占职工总数的百分比

### 3.6 排序功能（菜单形式）

8. 用菜单形式提供以下与排序相关的功能：
   - 列出最早到厂的职工的编号和姓名
   - 按出生日期的升序列输出全体职工的基本信息
   - 按出生日期的降序列出全体女职工的基本信息

**提示：** 出生日期的内部存储形式可用一个数字字符串表示，如"19850211"表示出生日期为1985年2月11日

### 3.7 数据存储要求

9. 当日出勤情况文件使用后，要先将全体数据添加到月逐日出勤情况文件 `MonthPunchIn`

### 3.8 登录和安全要求

10. 具有登录功能和密码加密功能

### 3.9 数据验证要求

11. 具有纠错功能，对手机号码和出生日期等信息要有正确性检查

## 四、数据结构设计

### 4.1 职工基本信息结构（EmpBasic）

```c
struct Employee {
    char id[20];        // 职工编号
    char name[20];      // 姓名
    char sex[10];       // 性别
    char birth_date[12]; // 出生日期（格式：19850211）
    char level[20];     // 职务级别
    char department[30]; // 所在部门
};
```

### 4.2 职工月出勤情况结构（Attendance）

```c
struct Attendance {
    char id[20];        // 职工编号
    int late_count;      // 迟到次数
    int leave_days;      // 请假天数
    int absent_days;     // 旷工天数
};
```

### 4.3 当日出勤记录结构（PunchIn）

```c
struct PunchRecord {
    char id[20];        // 职工编号
    int hour;          // 时
    int minute;        // 分
    int second;        // 秒
};
```

## 五、计算逻辑

### 5.1 迟到次数计算

```
到达时间 = 时 * 3600 + 分 * 60 + 秒
标准时间 = 8 * 3600 = 28800

迟到时间 = 到达时间 - 标准时间

如果 迟到时间 > 0:
    迟到分钟数 = 迟到时间 / 60
    迟到次数 = ceil(迟到分钟数 / 10)
```

### 5.2 旷工判断

```
旷工条件（满足其一即为旷工）：
1. 当日出勤文件中不出现该职工编号
2. 迟到时间 >= 3600（迟到1小时及以上）
```

### 5.3 允许请假天数计算

```
allowed_leave_days = 0

如果 性别 == "女":
    allowed_leave_days += 3

如果 性别 == "男" 且 年龄 >= 55:
    allowed_leave_days += 2

如果 当前月份 == 生日月份:
    allowed_leave_days += 1
```

### 5.4 年龄计算

```
current_year = 当前年份
current_month = 当前月份
current_day = 当前日期

birth_year = 出生日期的前4位
birth_month = 出生日期的5-6位
birth_day = 出生日期的7-8位

age = current_year - birth_year

if current_month < birth_month or (current_month == birth_month and current_day < birth_day):
    age -= 1
```

## 七、文件格式规范

```
编号 姓名 性别 出生日期 职务级别 所在部门
E001 张三 男 19850211 中级 技术部
E002 李四 女 19900305 高级 财务部
```

### 7.2 Attendance.txt（月出勤情况）

```
编号 迟到次数 请假天数 旷工天数
E001 2 1 0
E002 0 3 1
```

### 7.3 punchIn.txt（当日出勤记录）

```
编号 时 分 秒
E001 7 55 30
E002 8 5 45
```

**注意**：文件名应为 `punchIn.txt`

### 7.4 MonthPunchIn.txt（月逐日出勤）

```
编号 时 分 秒
E001 7 55 30
E001 8 0 15
...
```

## 八、开发注意事项

1. **编码问题**：所有数据文件使用 GB2312/GBK 编码
2. **日期处理**：出生日期使用8位数字字符串（YYYYMMDD）便于比较
3. **整数计算**：迟到时间、迟到次数等使用整数计算
4. **边界条件**：注意10分钟的边界（10分钟以下1次，10-20分钟2次）
5. **年龄判断**：年龄按周岁计算，未到生日不计入
6. **数据持久化**：修改后及时写入文件
7. **错误处理**：对不存在的职工编号、错误的数据格式进行提示
8. **界面友好**：使用菜单形式，每步操作后返回主菜单
