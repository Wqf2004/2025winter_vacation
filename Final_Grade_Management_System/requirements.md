# 期末成绩管理系统 - 详细需求规格说明书

## 一、项目背景

学生在期末考试结束后，任课老师把学生成绩单交教学办公室的教务员。教务员汇总后把学生成绩录入计算机，并记入学生成绩文件中。对于学生来说，还有一个学生基本情况文件记载着学生的学号、姓名、性别、出生日期、入学日期和奖罚等信息。

假设学生成绩文件由学号、计算机成绩、高等数学成绩、外语成绩、体育成绩和平均成绩组成。

## 二、业务规则

### 2.1 奖励规定

**期末成绩奖励（一次性奖励）：**
- 平均成绩85分以上（含85分）至90分（不含90分）：每人奖励400元
- 平均成绩90分以上（含90分）至95分（不含95分）：每人奖励800元
- 平均成绩95分以上（含95分）：每人奖励1200元

**奖励记录：**
- 将受奖次数记入学生基本情况文件中

### 2.2 补考与留级规定

**不及格处理：**
- 期末成绩不及格的学生应通知补考
- 计算机、高等数学和外语三门功课均不及格应发出通知直接留级

**留级与退学：**
- 补考、留级的信息要记入学生基本情况文件中
- 若留级次数达到2次，通知退学
- 若补考次数达到8次，通知退学

### 2.3 成绩等级划分

**成绩等级：**
- 优：90分以上
- 良：80-89分
- 中：70-79分
- 及格：60-69分
- 不及格：不足60分

### 2.4 数据文件

- `StudentBasic` - 学生基本情况文件（学号、姓名、性别、出生日期、入学日期、受奖次数、补考次数、留级次数）
- `StudentGrade` - 学生成绩文件（学号、计算机成绩、高等数学成绩、外语成绩、体育成绩、平均成绩）
- `password` - 登录密码文件

## 三、系统功能要求

### 3.1 成绩录入要求

1. 一次录入一个学生的全体成绩并计算平均分
2. 录入后就存入学生成绩文件中
3. 学生总人数在100之内
4. 学生基本情况文件和学生成绩文件均应设计为等长记录的文本文件

### 3.2 查询功能（菜单形式）

程序要用菜单形式提供以下查询功能：

1. **按学号查询学生基本情况**
   - 查询学号、姓名、性别、出生日期、入学日期
   - 查询受奖次数、补考次数、留级次数

2. **按学号查询学生成绩**
   - 查询各科成绩和平均成绩
   - 包含计算机、高等数学、外语、体育成绩

3. **列出期末成绩一览表**
   - 含学号、姓名、各科成绩和平均成绩

4. **列出受奖情况**
   - 含受奖学生的学号、姓名、各科成绩、平均成绩、受奖金额

### 3.3 排序功能（菜单形式）

1. **按平均成绩从高到低排序输出成绩一览表**
   - 包含学号、姓名、各科成绩和平均成绩

### 3.4 统计功能（菜单形式）

程序要用菜单形式提供以下统计功能：

1. **计算每门功课的平均分**
   - 计算机、高等数学、外语、体育四门课程的平均分

2. **求每门功课的最高分和最低分**
   - 每门课程的最高分
   - 每门课程的最低分

3. **统计每门功课各级别的人数**
   - 优（90分以上）
   - 良（80-89分）
   - 中（70-79分）
   - 及格（60-69分）
   - 不及格（不足60分）

4. **统计留级人数**
   - 留级学生总数

5. **统计退学人数**
   - 退学学生总数

### 3.5 登录功能

1. 具有登录功能
2. 具有密码加密功能

### 3.6 数据验证功能

1. 具有纠错功能
2. 对手机号码要有正确性检查
3. 对出生日期信息要有正确性检查

## 四、数据结构设计

### 4.1 学生基本情况结构（StudentBasic）

```c
struct StudentBasic {
    char id[20];       // 学号
    char name[20];     // 姓名
    char sex[4];       // 性别
    char birth_date[9]; // 出生日期（格式：YYYYMMDD）
    char enroll_date[9];// 入学日期（格式：YYYYMMDD）
    int award_count;    // 受奖次数
    int makeup_count;   // 补考次数
    int retain_count;   // 留级次数
};
```

### 4.2 学生成绩结构（StudentGrade）

```c
struct StudentGrade {
    char id[20];          // 学号
    float computer_score;  // 计算机成绩
    float math_score;     // 高等数学成绩
    float english_score;   // 外语成绩
    float pe_score;       // 体育成绩
    float average_score;   // 平均成绩
};
```

## 五、计算逻辑

### 5.1 平均成绩计算

```
平均成绩 = (计算机成绩 + 高等数学成绩 + 外语成绩 + 体育成绩) / 4
```

### 5.2 奖励金额计算

```
如果 85 <= 平均成绩 < 90:
    奖励金额 = 400元
否则如果 90 <= 平均成绩 < 95:
    奖励金额 = 800元
否则如果 平均成绩 >= 95:
    奖励金额 = 1200元
否则:
    奖励金额 = 0元
```

### 5.3 补考判断

```
如果 计算机 < 60 或 高等数学 < 60 或 外语 < 60 或 体育 < 60:
    通知补考
    补考次数 += 1
```

### 5.4 留级判断

```
如果 计算机 < 60 且 高等数学 < 60 且 外语 < 60:
    通知留级
    留级次数 += 1
```

### 5.5 退学判断

```
如果 留级次数 >= 2:
    通知退学

如果 补考次数 >= 8:
    通知退学
```

## 六、开发要求

### 6.1 数据结构

1. 为统计、查询方便，应建立一个一维数组Students
2. 该数组应含有学号、姓名、各科成绩和平均分等各项

### 6.2 界面要求

1. 各项功能应用菜单形式选择执行
2. 用户界面友好，操作简便

### 6.3 编码规范

- 使用 4 空格缩进
- 每行最大长度 100 字符
- 使用单引号而不是双引号（除非必要）
- 在语句末尾添加分号
- 函数上方必须有注释
- 复杂逻辑添加行内注释解释
- 注释使用中文

### 6.4 编码规范（文件编码）

- 含中文的文件使用 `gb2312` 编码
- 不含中文的文件使用 `utf-8` 编码
