# 学生信息管理系统 - 需求规格说明书

## 一、项目背景

学生信息管理系统是一个用于管理学生基本信息和课程成绩的综合管理系统。系统采用文本文件存储数据，支持数据录入、查询、排序、删除等核心功能，并提供三种技术实现版本：C语言控制台版、PyQt5桌面版和Streamlit网页版。

## 二、项目概述

### 系统目标
- 管理学生基本信息（学号、姓名、性别、宿舍号、电话号码）
- 管理学生课程成绩（课程编号、课程名称、学分、平时成绩、实验成绩、卷面成绩、综合成绩）
- 提供灵活的查询功能
- 支持数据排序和删除
- 提供登录验证和权限控制

### 版本规划
1. **C语言控制台版**：采用命令行界面，适合Windows/Linux环境
2. **PyQt5桌面版**：采用图形化界面，提供更好的用户体验
3. **Streamlit网页版**：采用Web界面，支持浏览器访问

## 三、数据结构设计

### 3.1 学生基本信息结构（a.txt）

```
学号 姓名 性别 宿舍号码 电话号码
```

**字段说明**：
- 学号：唯一标识，字符串类型
- 姓名：学生姓名
- 性别：男/女
- 宿舍号码：格式如"3-405"，表示3号楼405室
- 电话号码：11位手机号码

**示例**：
```
20230101 张三 男 3-401 13800138001
20230102 李四 男 3-401 13800138002
20230103 王五 女 5-301 13900139001
```

### 3.2 学生成绩信息结构（b.txt）

```
学号 课程编号 课程名称 学分 平时成绩 实验成绩 卷面成绩 综合成绩 实得学分
```

**字段说明**：
- 学号：对应学生基本信息
- 课程编号：课程唯一标识
- 课程名称：课程名称
- 学分：课程学分（1-5分）
- 平时成绩：平时成绩（0-100分）
- 实验成绩：实验成绩（0-100分），-1表示无实验课程
- 卷面成绩：卷面成绩（0-100分）
- 综合成绩：程序自动计算（平时×30% + 实验×15% + 卷面×70%）
- 实得学分：程序自动计算（按等级学分制）

**示例**：
```
20230101 CS101 计算机基础 4 85 90 88 88.35 3.2
20230101 CS102 数据结构 4 78 85 82 82.15 3.0
```

## 四、功能需求

### 4.1 数据录入功能

**需求描述**：
- 录入学生成绩信息到b.txt文件
- 只需录入7个字段：学号、课程编号、课程名称、学分、平时成绩、实验成绩、卷面成绩
- 综合成绩和实得学分由程序自动计算

**实现要求**：
1. 验证学号是否存在于a.txt中
2. 自动计算综合成绩：平时成绩×30% + 实验成绩×15% + 卷面成绩×70%
3. 自动计算实得学分（等级学分制）：
   - 综合成绩 ≥ 90：学分 × 1.0
   - 80 ≤ 综合成绩 < 90：学分 × 0.8
   - 70 ≤ 综合成绩 < 80：学分 × 0.75
   - 60 ≤ 综合成绩 < 70：学分 × 0.6
   - 综合成绩 < 60：学分 × 0（即0分）
4. 支持无实验课程（输入-1表示无实验成绩，计算时实验成绩部分不计入）
5. 验证成绩输入范围（0-100）

### 4.2 查询功能

**需求描述**：
提供多种查询方式，支持按学号、姓名、宿舍号查询学生信息和成绩。

**实现要求**：
1. **智能查询**：自动判断输入是学号还是姓名
2. **按学号查询**：查询学生基本信息
3. **按姓名查询**：查询学生基本信息
4. **按宿舍号查询**：查询同一宿舍的所有学生信息
5. **按学号查询成绩**：查询学生所有课程成绩，并统计总学分
6. **手机号码格式验证**：
   - 必须为11位
   - 必须以1开头，第二位为3-9
   - 必须全为数字

### 4.3 删除功能

**需求描述**：
删除学生时，同时删除该学生的基本信息和所有成绩记录。

**实现要求**：
1. 按学号删除学生信息
2. 删除前显示学生信息供确认
3. 同时删除a.txt中的基本信息和b.txt中的所有成绩记录
4. 使用临时文件确保数据安全

### 4.4 排序功能

**需求描述**：
支持按综合成绩或实得学分进行升序或降序排序。

**实现要求**：
1. 按综合成绩升序排列
2. 按综合成绩降序排列
3. 按实得学分升序排列
4. 按实得学分降序排列
5. 排序结果以表格形式显示，包含学号、姓名、课程编号、课程名称、学分、综合成绩

### 4.5 界面要求

**需求描述**：
采用文本菜单界面，如果可能采用图形菜单界面更好。

**实现要求**：
1. 美化的文本菜单界面
2. 使用`conio.h`的`getch()`实现按键响应
3. 使用`windows.h`的`Sleep()`实现延时
4. 使用`system("cls")`实现清屏
5. 菜单边框美化，提升用户体验
6. 每个操作完成后按任意键返回上一级菜单

### 4.6 登录功能

**需求描述**：
具有登录功能和密码加密功能。

**实现要求**：
1. 登录验证系统（3次尝试机会）
2. 密码输入时显示星号（*）保护隐私
3. 首次使用自动创建默认密码（admin123）
4. 密码保存在单独的password.txt文件中
5. 密码错误提示剩余尝试次数
6. 登录失败自动退出系统

### 4.7 数据验证功能

**需求描述**：
具有纠错功能，对手机号码等信息要有正确性检查。

**实现要求**：
1. 手机号码格式验证
2. 验证手机号码长度（必须为11位）
3. 验证手机号码格式（必须以1开头，第二位为3-9）
4. 验证手机号码是否全为数字
5. 在查询学生信息时显示手机号码格式验证结果（有效/无效）

## 五、技术规范

### 5.1 编码规范

**C语言版本**：
- 所有数据文件使用 GB2312/GBK 编码
- C语言源代码文件统一使用 UTF-8 编码
- 编译时指定编码：`-finput-charset=UTF-8 -fexec-charset=GBK`

**Python版本**：
- 所有Python文件使用 UTF-8 编码
- 数据文件使用 GBK 编码读取

### 5.2 文件编码问题

**问题说明**：
C语言的运行程序调用cmd.exe，Windows的cmd默认编码为936（GB2312/GBK）。VS Code默认文件编码为UTF-8，生成的可执行文件也是UTF-8编码的，但是系统编码为GB2312，所以程序中的中文在显示的时候出现了问题。

**解决方案**：
未来只要是与C语言编写的代码文件，包括C语言写入读出的文件统一使用GB2312中文编码格式。

## 八、核心算法

### 8.1 综合成绩计算

```
综合成绩 = 平时成绩 × 0.3 + 实验成绩 × 0.15 + 卷面成绩 × 0.7
```

**特殊情况**：
- 如果实验成绩为-1（无实验课程）：
  ```
  综合成绩 = 平时成绩 × 0.3 + 卷面成绩 × 0.7
  ```

### 8.2 实得学分计算（等级学分制）

```
if 综合成绩 ≥ 90:
    实得学分 = 学分 × 1.0
elif 综合成绩 ≥ 80:
    实得学分 = 学分 × 0.8
elif 综合成绩 ≥ 70:
    实得学分 = 学分 × 0.75
elif 综合成绩 ≥ 60:
    实得学分 = 学分 × 0.6
else:
    实得学分 = 0
```

### 8.3 手机号码验证

```
验证规则：
1. 长度必须为11位
2. 必须以1开头
3. 第二位必须是3-9
4. 所有字符必须是数字
```

**实现代码**：
```c
int validatePhoneNumber(const char *phone) {
    // 检查长度
    if (strlen(phone) != 11) {
        return 0;
    }

    // 检查是否全为数字
    for (int i = 0; i < 11; i++) {
        if (!isdigit(phone[i])) {
            return 0;
        }
    }

    // 检查第一位
    if (phone[0] != '1') {
        return 0;
    }

    // 检查第二位
    if (phone[1] < '3' || phone[1] > '9') {
        return 0;
    }

    return 1;
}
```
